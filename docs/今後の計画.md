# ガチャカウンター 今後の計画

## 1. X (Twitter) ポスト投稿機能

### 概要
ガチャ結果をワンクリックで X にポストできる機能を追加する。

### 想定する投稿内容
- キャラ名・獲得までの回数・累計回数
- ゲーム名やハッシュタグ（URLパラメータで設定可能にする）
- 例: `【ガチャ結果】キャラ名 を 42回 で獲得！(累計 215回目) #ゲーム名 #ガチャカウンター`

### 実装方針
- Twitter Web Intent URL を使用（OAuth 不要、ブラウザでXの投稿画面を開く）
  - `https://twitter.com/intent/tweet?text=...`
- OBSブラウザソース内からは別ウィンドウが開けないため、**Next.js版 (Web公開版)** での実装を優先する
- 現行の静的HTML版では `window.open()` をトリガーするボタンを設置（OBSでは動作しない可能性あり）

### 必要な設定項目 (URLパラメータ化候補)
| パラメータ | 説明 | 例 |
|---|---|---|
| `gameName` | ゲーム名 | `ブルアカ` |
| `hashtags` | ハッシュタグ（カンマ区切り） | `ブルーアーカイブ,ガチャ` |
| `postTemplate` | 投稿文テンプレート | カスタム文字列 |

## 2. 広告収入による収益化

### 概要
Web公開版（Next.js版）にて広告を掲載し、収益化を図る。

### 想定する広告手段

| 手段 | 特徴 | 備考 |
|---|---|---|
| Google AdSense | 審査あり・高単価 | PV数が一定以上になってから申請 |
| Amazonアソシエイト | ゲーム関連グッズ紹介 | 記事ページを設ける場合に有効 |
| アフィリエイト (A8等) | ゲーム課金サービス紹介 | ガチャ系ユーザーと相性が良い |

### 収益化に向けたロードマップ

1. **Next.js版 Web公開** - Vercel 等でホスティング
2. **SEO・SNS流入の確保** - X投稿機能でユーザーが自然に拡散
3. **PV数の確認** - Google Analytics 導入
4. **AdSense 申請** - 規約を満たすコンテンツ量・PV数が揃ったタイミングで申請
5. **広告配置の最適化** - UX を損なわない位置に配置

### 注意事項
- OBSオーバーレイ版（静的HTML）には広告を掲載しない（配信画面に映り込むため）
- 広告はあくまで Web公開版のみに限定する
