# 01 - HTML版 Xポスト機能 設計と全体像

## 概要

「獲得!」ボタンで記録したガチャ結果を、ワンクリックで X (Twitter) にポストできるボタンを追加する。
OAuthによるAPI認証は不要で、X公式の **Twitter Web Intent** 機能を利用する。

---

## Twitter Web Intent とは

以下のURLをブラウザで開くと、Xの投稿画面がテキスト入力済みの状態で開く仕組み。

```
https://twitter.com/intent/tweet?text=投稿したいテキスト(URLエンコード済み)
```

- アプリ登録・APIキー不要
- ユーザー自身がXにログインしていればそのまま投稿できる
- `window.open()` で別ウィンドウとして開く

### 生成されるツイート例

```
【ブルアカ ガチャ結果】
ホシノ を 47回 で獲得！
累計 183回目

#ガチャカウンター #ブルアカ #ブルーアーカイブ
```

---

## OBS ブラウザソースでの制限事項

| 環境 | 動作 |
|---|---|
| 通常のブラウザ | `window.open()` で投稿画面が別タブ/ウィンドウで開く ✅ |
| OBS ブラウザソース | セキュリティ制限により `window.open()` は動作しない ❌ |

**対応方針:**
- OBSオーバーレイ配信中はXポスト機能は使用しない
- 配信終了後、通常のブラウザ (Chrome/Edge等) で `index.html` を開いてポストする
- ボタン自体は表示するが、OBS上では機能しない旨をマニュアルに記載する

---

## 実装フロー

```
ユーザーが「Xにポスト」ボタンをクリック
    ↓
buildTweetText() - 最新の獲得履歴からツイート文を生成
    ↓
getUrlParams() - URLパラメータからゲーム名・ハッシュタグを取得
    ↓
encodeURIComponent() でURLエンコード
    ↓
Twitter Web Intent URL を組み立て
    ↓
window.open() で投稿画面を別ウィンドウで開く
```

## URLパラメータ仕様

OBSのブラウザソースURL、または通常ブラウザのアドレスバーで設定する。

| パラメータ名 | 型 | 説明 | 例 |
|---|---|---|---|
| `gameName` | 文字列 | ゲーム名（ツイート文とハッシュタグに使用） | `ブルアカ` |
| `hashtags` | 文字列 | 追加ハッシュタグ（カンマ区切り、`#`は不要） | `ブルーアーカイブ,ガチャ` |

### 設定例

```
file:///C:/Users/miz32/Desktop/ガチャカウンター/index.html?gameName=ブルアカ&hashtags=ブルーアーカイブ,ガチャ
```
