# 06 - フォルダ分離 設計と方針

## 概要

OBS配信オーバーレイ用と、ブラウザで開く用途のファイルをフォルダごと分離する。
それぞれ目的が異なるため、機能・デザインを独立して管理する。

---

## フォルダ構成

```
ガチャカウンター/
├── obs/                    ← OBS ブラウザソース用
│   ├── index.html
│   ├── css/
│   │   └── style.css
│   └── js/
│       └── app.js
│
├── browser/                ← ブラウザで開く用（Xポスト・広告あり）
│   ├── index.html
│   ├── css/
│   │   └── style.css
│   └── js/
│       └── app.js
│
└── docs/                   ← ドキュメント類（変更なし）
```

> ルート直下の `index.html` / `css/` / `js/` は残しても削除してもよい。
> 今後は `obs/` と `browser/` を正として管理する。

---

## 各バージョンの機能比較

| 機能 | OBS版 | ブラウザ版 |
|---|---|---|
| カウント・履歴・サマリー | ✅ | ✅ |
| 凸数表示 | ✅ | ✅ |
| localStorage 自動保存 | ✅ | ✅ |
| Xポストボタン | ❌ 非搭載 | ✅ |
| URLパラメータ（gameName/hashtags） | ❌ 不要 | ✅ |
| 広告 | ❌ 非搭載 | ✅ |
| 透明背景（OBS向け） | ✅ | ❌ 不要 |

---

## localStorage の扱い

OBSブラウザソースとブラウザはlocalStorageを共有しない（別々に管理される）。
そのため、**OBS版とブラウザ版はそれぞれ独立したデータを持つ**。

### 運用フロー

```
配信中
  → obs/index.html を OBS ブラウザソースに登録して操作・記録

配信後（Xポスト・結果確認）
  → browser/index.html をブラウザで開く
  → OBS版とは別のlocalStorageのため、記録は引き継がれない
  → 配信後に結果を手入力するか、ブラウザ版を配信中も並行して操作する
```

### 推奨運用（localStorageを共有しない問題の回避策）

ブラウザ版をブラウザで開きながら配信する。
OBSには「ウィンドウキャプチャ」でブラウザのウィンドウを取り込む方式にすると、
ブラウザのlocalStorageで統一でき、配信後そのままXポストが可能。
