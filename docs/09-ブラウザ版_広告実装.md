# 09 - ブラウザ版 広告実装

## 概要

`browser/` フォルダのページに広告を設置する。
広告手段は2段階で対応する。

| 段階 | 手段 | 条件 |
|---|---|---|
| Phase 1 | アフィリエイトリンク（バナー画像） | ローカルファイル (`file://`) でも今すぐ動作 |
| Phase 2 | Google AdSense | GitHub Pages 等でホスティング後に申請 |

---

## Phase 1: アフィリエイトリンク

### 特徴

- `file://` で開くローカルHTMLでも動作する
- バナー画像は各ASP（アフィリエイトサービスプロバイダ）が提供するHTMLタグをそのまま貼り付ける
- おすすめASP: **A8.net**（ゲーム課金サービス系の案件が豊富）、**Amazonアソシエイト**

### A8.net での手順

1. [A8.net](https://www.a8.net/) に登録・ログイン
2. 「広告プログラム」から掲載したい案件を検索（例：「ゲーム課金」「スマホゲーム」）
3. 提携申請→承認後「広告タグ取得」からバナーのHTMLタグをコピー
4. `browser/index.html` の `<div class="ad-area">` 内に貼り付ける

### Amazonアソシエイトでの手順

1. [Amazonアソシエイト](https://affiliate.amazon.co.jp/) に登録・審査通過
2. 「リンクを作成」からゲーム関連商品（スマホゲームの課金ギフトカード等）のバナーHTMLを取得
3. `browser/index.html` の `<div class="ad-area">` 内に貼り付ける

### 貼り付け後の browser/index.html（広告エリア部分）

```html
    <!-- 広告エリア -->
    <div class="ad-area" id="adArea">
      <!-- ここに A8.net / Amazonアソシエイト のバナータグを貼り付ける -->
      <!-- 例: -->
      <!-- <a href="https://px.a8.net/svt/ejp?a8mat=..." target="_blank">
             <img src="https://www22.a8.net/svt/bgt?aid=...&wid=..." width="300" height="250" alt="">
           </a> -->
    </div>
```

## Phase 2: Google AdSense

### 前提条件

AdSense は `file://` では動作しない。
**GitHub Pages でホスティングし、`https://` でアクセスできる状態**にしてから申請する。

---

### GitHub Pages でのホスティング手順

#### 1. GitHubリポジトリを作成

1. [GitHub](https://github.com/) にログイン
2. 右上「+」→「New repository」
3. Repository name: `gacha-counter`（任意）
4. Public に設定（GitHub Pages は Public リポジトリなら無料）
5. 「Create repository」をクリック

#### 2. browser/ フォルダの内容をリポジトリにアップロード

リポジトリのルートに `browser/` の中身（`index.html`, `css/`, `js/`）を配置する。

```
gacha-counter/          ← リポジトリルート
├── index.html          ← browser/index.html の内容
├── css/
│   └── style.css
└── js/
    └── app.js
```

ローカルのgitで操作する場合:

```bash
cd /mnt/c/Users/miz32/Desktop/ガチャカウンター
git subtree push --prefix browser origin main
```

または `browser/` 内容を手動でGitHubのUIからアップロードしてもよい。

#### 3. GitHub Pages を有効化

1. リポジトリページの「Settings」→「Pages」
2. Source: 「Deploy from a branch」
3. Branch: `main` / `/ (root)` を選択して「Save」
4. しばらく待つと `https://ユーザー名.github.io/gacha-counter/` でアクセスできるようになる

---

### Google AdSense 申請手順

#### 1. AdSense に登録

1. [Google AdSense](https://www.google.com/adsense/) にアクセス
2. Googleアカウントでログイン
3. サイトURLに GitHub Pages の URL を入力
   例: `https://ユーザー名.github.io/gacha-counter/`
4. 申請を送信

#### 2. サイト審査の要件（事前に確認）

| 要件 | 内容 |
|---|---|
| コンテンツ量 | オリジナルコンテンツがある程度必要（ツールのみだと薄いと判断される場合あり） |
| プライバシーポリシー | ページ内またはリンクで掲載が必要 |
| お問い合わせ | 連絡手段の記載が望ましい |
| PV数 | 明確な基準はないが、一定のアクセスがあると承認されやすい |

#### 3. 審査通過後：AdSenseコードを browser/index.html に追加

審査通過後、AdSenseの管理画面から「自動広告」または「ディスプレイ広告」のコードを取得する。

**自動広告の場合**（`<head>` 内に1行追加するだけ）:

```html
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>ガチャカウンター</title>
  <link rel="stylesheet" href="css/style.css">
  <!-- Google AdSense -->
  <script async src="https://pagead2.googlesyndication.com/pagead/js/adsbygoogle.js?client=ca-pub-XXXXXXXXXX"
     crossorigin="anonymous"></script>
</head>
```

`ca-pub-XXXXXXXXXX` の部分はAdSense管理画面で発行されるパブリッシャーIDに置き換える。

**ディスプレイ広告を広告エリアに配置する場合**（`.ad-area` 内に追加）:

```html
    <!-- 広告エリア -->
    <div class="ad-area" id="adArea">
      <ins class="adsbygoogle"
           style="display:block"
           data-ad-client="ca-pub-XXXXXXXXXX"
           data-ad-slot="XXXXXXXXXX"
           data-ad-format="auto"
           data-full-width-responsive="true"></ins>
      <script>
           (adsbygoogle = window.adsbygoogle || []).push({});
      </script>
    </div>
```

`data-ad-slot` の値はAdSense管理画面の「広告ユニット」から取得する。

## ホスティング先の検討

### GitHub Pages の商用利用について

GitHub Pages は無料で利用できるが、利用規約に以下の制限がある。

> 「商用目的のウェブホスティングサービスとして使用してはならない」

AdSense を貼る程度であれば実態として多くの開発者が利用しているが、
**厳密には商用利用がグレーゾーン**であるため、本格的な収益化には向かない。

---

### 代替ホスティングサービス（商用利用が明示的に許可されている無料枠あり）

| サービス | 無料枠 | 商用利用 | 備考 |
|---|---|---|---|
| **Vercel** | 帯域・ビルド回数十分 | ✅ 明示許可 | Next.js との相性が最良。静的HTMLも対応 |
| **Netlify** | 月100GB帯域 | ✅ 明示許可 | 静的HTMLのデプロイが簡単 |
| **Cloudflare Pages** | 帯域ほぼ無制限 | ✅ 明示許可 | CDNが高速。無料枠が最も寛大 |

### 推奨: Vercel

- Next.js版の制作を予定しているため、**Vercel に一本化すると管理が楽**
- 静的HTMLの `browser/` も Next.js 版と同じリポジトリにまとめてデプロイ可能
- GitHub連携で `main` ブランチにプッシュするだけで自動デプロイされる

### Vercel でのデプロイ手順（概要）

1. [Vercel](https://vercel.com/) に GitHub アカウントでサインアップ
2. 「New Project」→ GitHubのリポジトリを選択
3. Framework Preset: 静的HTMLの場合は「Other」を選択
4. Root Directory: `browser/` を指定
5. 「Deploy」をクリック
6. `https://プロジェクト名.vercel.app/` でアクセス可能になる
7. カスタムドメインを取得した場合は Settings > Domains から設定可能
